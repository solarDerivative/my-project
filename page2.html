EDIT: Sandbox

  • YES! I FIGURED IT OUT: I was looking over my two functions, plusDivs(n) and showDivs(n), wondering why "n" was behaving the way it was. And then it occurred to me that I don't ever "declare" n as a variable, or at least not in the "traditional" sense (right? excuse my misunderstanding of terminology, I'm still getting there). So then, n might not be static - not even referring to the same thing - across these two functions.
  • Parameters: (I promise I knew and understood these terms back when I learned them in high school...) Anyway, once that thought occurred to me, I wanted to find out more about the properties of whatever it is you're putting in those function's parentheses. Found here:
    • formal parameter: the identifier used in a method to stand for the value that is passed into the method by a caller
    • actual parameter: the actual value that is passed into the method by a caller
  • (n)'s All the Way Down (alt. A Tale of Two (n)'s: Identifier used IN a method to stand for the value THAT IS PASSED INTO the method. I had been treating the n of plusDivs(n) and the n of showDivs(n) as the same thing. By changing showDivs(n) to showDivs(k) and all instances of n inside that method to k, I was able to find out for sure that they were completely different things. Mystery solved!

First Speedbump
6/29/16 Notes

 So my next "tutorial-less upgrade project" was to change the "Pictures" widget on the Sandbox to where, instead of clicking text to show an image, it would be one of those shideshows where you can click to show the next or previous image. My process was to 1) research the basic functioning Javascript to make it work, 2) edit the HTML and CSS so I liked the soon-to-be end product, then 3) delete the Javascript then re-write it in a way that makes sense to me.

 I got it mostly done, but needed to cheat for some of the details. I added the details, and the slideshow worked just fine. But looking back over the code as a whole, I didn't fully understand what was happening on every single line. I commented-out some bits and pieces so that I could see exactly how each function and loop was working at any given time. And that was good.

 Now, I've hit a point where I have no idea what is going on. The way the code is commented out, on the slideshow (comprised of 3 images) you can click the Next button as many times as you want, and after the 3rd/last image in the array nothing will appear, and n += 1 each time, so n can go as high as you want. On the flip side, when you click the Previous button, it will cycle through the images without returning nothing (intended behavior), and will -1 each time, but when it should hit -3, it resets to 0 instead. (0, -1, -2, 0, -1, -2, 0, etc)

 The way I've described it, it seems to make sense: Previous cycles through the array correctly, so n gets reset when you go from the (1 2 3) first position to the (1 2 3) last position of the array. Next does NOT cycle through the array correctly, it will stop showing images after you (1 2 3) move past the last image.

 What I'm not getting is, why is it n that's being reset like this? As far as I can tell from my code, n -= 1 when Previous is clicked and n += 1 whenever Next is clicked. There's nothing in my code that I can see trying to change or set the value of n outside of its (+= 1 / -= 1) behavior. I have variable slideIndex that is supposed to be the variable that is acted upon and reset whenever the user cycles through the array (or so I thought).

 This one will take some figuring out! I am not moving on until this code makes absolute sense to me.
